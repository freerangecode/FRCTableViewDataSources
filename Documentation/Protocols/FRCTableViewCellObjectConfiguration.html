FRCTableViewCellObjectConfiguration Protocol Reference

Conforms to NSObject
Declared in FRCTableViewCell.h

Overview

While most cells used in the FRCTableViewDataSource system should be subclasses of FRCTableViewCell, you might want to use a third party cell that cannot be a subclass. You can still take advantage of the object configuration system by implementing this protocol.

In some cases, you may way to have a custom height for your cell. This class method allows us to determine how big a cell should be without having an instance. Use this method to perform calculations for the height of the cell.
